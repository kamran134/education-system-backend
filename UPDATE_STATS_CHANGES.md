# Обновление функции updateStats

## Описание изменений

Функция `updateStats` в сервисе статистики была полностью переписана с новым алгоритмом для определения лучших студентов месяца.

## Переименования

- Старая функция `updateStats` → `updateStatsOld` (сохранена для совместимости)
- Создана новая функция `updateStats` с улучшенным алгоритмом

## Новый алгоритм работы

### 1. Инициализация
- Получаем текущий месяц и год
- Находим все результаты студентов (`studentResult`) только за текущий месяц
- Используем поля `month` и `year` для фильтрации

### 2. Сброс предыдущих значений
- Обнуляем `studentOfTheMonthScore` и `republicWideStudentOfTheMonthScore` для всех результатов текущего месяца

### 3. Группировка результатов
- **По классам и районам**: для определения лучших в каждом классе каждого района
- **По классам**: для определения лучших в каждом классе по всей республике

### 4. Определение лучших по районам
- Для каждой группы "класс-район" находим максимальное значение `totalScore`
- Всем студентам с максимальным `totalScore` в своей группе присваиваем `studentOfTheMonthScore = 5`

### 5. Определение лучших по республике  
- Для каждого класса (по всем районам) находим максимальное значение `totalScore`
- Всем студентам с максимальным `totalScore` в своем классе по республике присваиваем `republicWideStudentOfTheMonthScore = 5`

## Пример работы

**Исходные данные:**
- 5 класс, район А: студенты с баллами 80, 90, 85
- 5 класс, район Б: студенты с баллами 75, 88
- 6 класс, район А: студенты с баллами 82, 95
- 6 класс, район Б: студенты с баллами 84, 87

**Результат:**
- **Лучшие по районам** (получают `studentOfTheMonthScore = 5`):
  - 5 класс, район А: студент с 90 баллами
  - 5 класс, район Б: студент с 88 баллами  
  - 6 класс, район А: студент с 95 баллами
  - 6 класс, район Б: студент с 87 баллами

- **Лучшие по республике** (получают `republicWideStudentOfTheMonthScore = 5`):
  - 5 класс: студент с 90 баллами (лучший среди всех районов в 5 классе)
  - 6 класс: студент с 95 баллами (лучший среди всех районов в 6 классе)

## Особенности реализации

- **Работа только с текущим месяцем**: функция обрабатывает только результаты за текущий календарный месяц
- **Массовые обновления**: использует `bulkWrite` для эффективности при большом количестве записей
- **Подробное логирование**: выводит детальную информацию о процессе выполнения
- **Обработка нескольких лидеров**: если несколько студентов имеют одинаковый максимальный балл, все получают награду

## Безопасность

- Старая функция сохранена как `updateStatsOld` для возможности отката
- Новая функция не влияет на другие поля результатов студентов
- Обработка ошибок с детальным логированием

## Применение

Функция вызывается через тот же API endpoint, что и раньше. Все существующие интеграции продолжат работать без изменений.
